開始前，先搞懂我們要做什麼

🎯 目標：知道等一下每個步驟在幹嘛，不會做到一半迷路。

做一個網站，其實就是把三樣東西接起來。用開一間店來比喻：

  1. GitHub = 設計圖倉庫。網站的「程式碼」放在這裡，我們會複製一份現成的設計圖變成你的。
  2. Supabase = 你的倉庫。娃娃的照片和資料都存在這裡。
  3. Vercel = 房東 + 店面。它把設計圖蓋成真的網站，給你一個網址讓大家逛。

這三個服務都有免費方案，個人收藏展示的用量完全不會超過免費額度，不用綁信用卡。

你需要準備的東西

  1. 一個 email 信箱（三個服務註冊都用它）
  2. 一台電腦（手機也能看網站，但設定過程用電腦操作比較不容易出錯）
  3. 一段不被打擾的 40 分鐘 ☕
等一下會請你抄下幾把「金鑰」（很長的亂碼）。建議先開一個記事本檔案，抄的時候貼在裡面，最後全部填完再刪掉。
1

註冊 GitHub 帳號

🎯 目標：有一個 GitHub 帳號。已經有的話直接跳到步驟 2。

  1. 打開 github.com/signup
  2. 輸入 email、設定密碼、取一個帳號名稱（之後你的網站網址會用到這個名稱，建議取短一點的英文）
  3. 完成信箱驗證，看到 GitHub 首頁就成功了
註冊過程如果出現問卷（問你是學生還是工程師之類的），隨便選或按 Skip 跳過都可以，不影響任何功能。
2

Fork：複製一份網站設計圖

🎯 目標：把 Collector 專案複製成你自己的。

「Fork」的意思就是：把別人的專案原封不動複製一份到你的帳號底下，之後它就完全是你的了，怎麼改都不會影響到原作者。

  1. 登入 GitHub 後，打開 github.com/Sallynium/Collector
  2. 點右上角的 Fork 按鈕
  3. 出現的頁面什麼都不用改，直接按綠色的 Create fork
  4. 幾秒後你會被帶到 github.com/你的帳號/Collector —— 這份就是你的了！
3

Supabase：建立你的雲端倉庫

🎯 目標：建好資料庫，抄下 3 把金鑰。這是最長的一步，慢慢來。

3-1. 註冊並建立專案

  1. 打開 supabase.com，點 Start your project，選擇用 GitHub 帳號登入（剛好步驟 1 辦好了！）
  2. 點 New project 建立新專案：
    • Name：隨意，例如 my-collection
    • Database Password：按 Generate 產生一組，抄到記事本（平常用不到，但遺失很麻煩）
    • Region：選 Northeast Asia (Tokyo)，離台灣最近、速度最快
  3. 按 Create new project，等 1–2 分鐘讓它開通

3-2. 蓋好倉庫的架子（執行 SQL）

倉庫開好了，但裡面還是空的，要先把「放娃娃資料的架子」蓋起來。下面這段指令會自動蓋好全部的架子，你只要複製 → 貼上 → 執行：

  1. 在 Supabase 左邊選單找到 SQL Editor（圖示像一張紙）
  2. 點下面的「複製 SQL」按鈕，把整段指令貼進編輯器
  3. 按右下角綠色的 Run，看到 Success. No rows returned 就完成了
📋 點我展開 SQL 指令（或直接按右邊的複製按鈕） 
-- ============================================================
-- Esports Doll Showcase — Supabase Schema
-- 在 Supabase Dashboard → SQL Editor 貼上執行即可
-- ============================================================

-- 系列（如：A 系列、2025 世界賽紀念系列）
create table if not exists series (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  description text,
  created_at timestamptz not null default now()
);

-- 收藏櫃 / 位置（如：A 櫃、客廳展示架）
create table if not exists cabinets (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  location text,
  created_at timestamptz not null default now()
);

-- 娃娃檔案
create table if not exists dolls (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  description text,
  series_id uuid references series(id) on delete set null,
  cabinet_id uuid references cabinets(id) on delete set null,
  created_at timestamptz not null default now()
);

-- 造型（一隻娃娃可有多張去背照片：預設、夏季穿搭、戰隊服…）
create table if not exists doll_variants (
  id uuid primary key default gen_random_uuid(),
  doll_id uuid not null references dolls(id) on delete cascade,
  name text not null default '預設造型',
  image_url text not null,
  sort_order int not null default 0,
  created_at timestamptz not null default now()
);

-- 虛擬收藏室場景
create table if not exists scenes (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  background_url text,
  layout jsonb not null default '[]'::jsonb,
  is_published boolean not null default false,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

-- ---------- Row Level Security ----------
-- 公開唯讀；寫入一律走伺服器端 service_role（繞過 RLS）
alter table series enable row level security;
alter table cabinets enable row level security;
alter table dolls enable row level security;
alter table doll_variants enable row level security;
alter table scenes enable row level security;

create policy "public read series" on series for select using (true);
create policy "public read cabinets" on cabinets for select using (true);
create policy "public read dolls" on dolls for select using (true);
create policy "public read variants" on doll_variants for select using (true);
create policy "public read published scenes" on scenes for select using (is_published = true);

-- ---------- Storage ----------
-- 建立公開圖片 bucket（去背 PNG 與場景背景圖）
insert into storage.buckets (id, name, public)
values ('doll-images', 'doll-images', true)
on conflict (id) do nothing;

create policy "public read doll-images"
on storage.objects for select
using (bucket_id = 'doll-images');

3-3. 抄下 3 把金鑰

金鑰就像倉庫的鑰匙，等一下要交給 Vercel，網站才有權限進倉庫拿資料。

  1. 左邊選單最下面點 Project Settings（齒輪圖示）→ API（或 API Keys）
  2. 把這 3 個值抄到記事本：
    • Project URL（長得像 https://xxxx.supabase.co）
    • anon public 金鑰（新版介面可能叫 publishable）
    • service_role 金鑰（新版介面可能叫 secret，要點一下眼睛圖示才會顯示）
service_role 金鑰 = 倉庫萬能鑰匙，絕對不要貼到任何公開的地方（社群、聊天群、GitHub）。它只會被填進 Vercel 的設定裡，其他地方都不該出現。
4

Vercel：把網站架起來

🎯 目標：網站上線，拿到你自己的網址。

4-1. 用 GitHub 帳號登入 Vercel

  1. 打開 vercel.com/signup
  2. 選 Hobby（免費方案），然後點 Continue with GitHub 授權登入

4-2. 匯入你的 Collector

  1. 進入後點 Add New… → Project
  2. 清單裡找到你剛剛 Fork 的 Collector，點 Import

4-3. 填入 4 個環境變數（最關鍵的一步！）

在部署設定頁找到 Environment Variables 區塊，把下面 4 組「名稱 + 值」一組一組加進去。名稱可以直接按複製按鈕，一個字都不能差：

名稱（照抄）值（填什麼）
NEXT_PUBLIC_SUPABASE_URL
 步驟 3-3 抄的 Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
 步驟 3-3 抄的 anon public 金鑰
SUPABASE_SERVICE_ROLE_KEY
 步驟 3-3 抄的 service_role 金鑰
ADMIN_PASSWORD
 自己發明一組管理密碼，之後登入後台用。別用你在其他網站用過的密碼

4-4. 部署！

  1. 按 Deploy，等 1–3 分鐘
  2. 看到滿天彩帶的 Congratulations! 就成功了 🎉
  3. 點畫面上的預覽圖或 Visit，你的網站網址長得像 https://collector-xxxx.vercel.app —— 把它存到書籤！
如果部署完打開網站看到「🔧 尚未完成初始設定」，代表環境變數沒填對，看常見問題第 1 條就能修好。
5

開始佈置你的收藏室

🎯 目標：上架第一隻娃娃、拼出第一個場景。

5-1. 登入管理後台

  1. 在你的網站網址後面加上 /login（例如 https://collector-xxxx.vercel.app/login）
  2. 輸入你在步驟 4-3 自己發明的 ADMIN_PASSWORD

5-2. 建立分類和第一隻娃娃

  1. 進入 庫存管理，先在「系列」和「收藏櫃」分頁各建至少一筆（例如系列「戰隊限定」、收藏櫃「A 櫃」）
  2. 回到「娃娃」分頁，新增你的第一隻娃娃，選好它的系列和櫃位

5-3. 上傳照片，讓 AI 幫你去背

  1. 點進剛建好的娃娃 → 新增造型 → 選一張娃娃照片（背景越乾淨效果越好）
  2. 等 AI 自動去背（第一次會下載模型，需要 1–2 分鐘，之後就快了）
  3. 如果 AI 誤刪了耳機或頭髮，用 🟢 綠色筆刷塗回來；多留了雜物就用 🔴 紅色筆刷擦掉
  4. 滿意後按儲存。一隻娃娃可以放很多造型（預設、夏季穿搭、戰隊服…），前台可以點擊切換

5-4. 拼出你的虛擬收藏室

  1. 後台進入 虛擬收藏室 → 建立場景、取個名字
  2. 上傳一張背景圖（書櫃照片、電競桌照片、或任何你喜歡的圖）
  3. 把娃娃一隻隻拖進場景，自由縮放、旋轉、調整前後順序
  4. 儲存 → 發布，訪客就能在 /scenes 頁看到

5-5. 分享給朋友 🎉

把你的網址（不含 /login）傳給朋友，他們就能瀏覽你的收藏、點娃娃切換造型、逛你拼的虛擬收藏室。後台只有你有密碼能進，不用擔心被亂改。

常見問題

1. 網站打開顯示「🔧 尚未完成初始設定」

代表 Vercel 沒讀到 Supabase 的設定。到 Vercel → 你的專案 → Settings → Environment Variables 檢查步驟 4-3 的 4 組變數：名稱有沒有打錯字、值有沒有多複製到空格。修正後到 Deployments 分頁，點最新那筆右邊的「…」→ Redeploy，環境變數改完一定要重新部署才會生效。

2. 忘記管理密碼了怎麼辦？

不用怕，密碼就存在 Vercel 的環境變數裡。到 Vercel → Settings → Environment Variables 找到 ADMIN_PASSWORD，可以直接看值，或改成新的（改完記得 Redeploy）。

3. 圖片上傳失敗 / 存檔一直轉圈圈

最常見的原因是步驟 3-2 的 SQL 沒有執行成功（架子沒蓋好）。回 Supabase 的 SQL Editor 把指令重新貼上執行一次——這段指令設計成重複執行也不會壞，放心按 Run。另外到左邊選單 Storage 確認有一個叫 doll-images 的 bucket。

4. 網站放了一陣子沒動，突然打不開 / 資料都不見了？

資料沒有不見！Supabase 免費方案的專案閒置 7 天會自動休眠。到 Supabase Dashboard 打開你的專案，按 Restore 按鈕，等幾分鐘就會全部回來。常常有朋友來逛的話就不會休眠。

5. 真的完全免費嗎？額度用得完嗎？

Vercel Hobby 方案和 Supabase 免費方案對個人展示站來說非常夠用：Supabase 免費給 500MB 資料庫 + 1GB 圖片空間，以去背後的 PNG 來算大約能放幾百張。AI 去背是在你自己的瀏覽器裡運算的，不花任何伺服器費用。兩個服務免費方案都不需要綁信用卡，所以就算真的用爆了也只會暫停，不會被扣錢。

6. 想換網址名稱（vercel.app 前面那段）

到 Vercel → 你的專案 → Settings → Domains，點 Edit 就能改成 你想要的名字.vercel.app（沒被別人用走的話）。

7. 原版 Collector 更新了，我要怎麼跟上？

打開你的 github.com/你的帳號/Collector，如果上方出現「This branch is X commits behind」，點旁邊的 Sync fork → Update branch，Vercel 會自動重新部署成新版。你的娃娃資料都在 Supabase，更新程式不會影響資料。
