先搞懂運作原理（很重要，只要 2 分鐘）

🎯 目標：理解「你的電腦 = 工作室，GitHub = 展場」這個模型，後面每一步都會很自然。

這個專案的設計哲學是：不用資料庫，你的收藏資料和照片就是專案資料夾裡的檔案。

  1. 在你的電腦上：開一個「管理後台」網頁，像逛電商後台一樣新增娃娃、AI 去背、拼場景。每次按下儲存，改的其實是資料夾裡的檔案（一個 JSON 清單 + 一堆去背 PNG）。
  2. 發布的時候：用 git push 把資料夾同步到 GitHub，GitHub 會自動把它組裝成一個漂亮的展示網站，任何人都能瀏覽。

這樣做的好處：

  1. 永遠免費 — 沒有資料庫帳號、沒有伺服器、沒有「免費額度用完」的問題
  2. 永不休眠 — 展示網站是純靜態網頁，朋友半夜點開也秒開
  3. 自帶時光機 — 每次發布都是一個 git 紀錄，改壞了隨時退回上一版

你需要準備的東西

  1. 一台電腦（Windows 或 Mac 都可以，本教學以 Windows 為主，Mac 對應之處會標註）
  2. 一個 email 信箱（註冊 GitHub 用）
  3. 初次設定約 60 分鐘；之後每次更新收藏只要 5 分鐘 ☕
教學裡所有灰黑色的「終端機區塊」都有複製按鈕，你不需要手打任何指令——複製、貼上、按 Enter，就這樣。
1

安裝兩個工具：Git 和 Node.js

🎯 目標：裝好 Git（同步工具）和 Node.js（讓後台在你電腦上跑起來的引擎），並確認能用。

1-1. 先學會開「終端機」

等一下的指令都在「終端機」裡執行——就是那個黑黑的打字視窗，別怕，我們只做複製貼上：

  1. Windows：對「開始」按鈕按右鍵 → 選「終端機」（或按 Win + X 再按 A）
  2. Mac：按 Cmd + 空白鍵，輸入 Terminal，按 Enter

1-2. 安裝 Git

  1. 到 git-scm.com/downloads 下載你系統的安裝檔
  2. 執行安裝，全部用預設值一直下一步就好（選項很多很嚇人，都不用改）
  3. 裝完重開終端機，貼上這行確認：
確認 Git 裝好了
git --version

看到類似 git version 2.50.0 的字樣就成功了（數字不同沒關係）。

1-3. 安裝 Node.js

  1. 到 nodejs.org 下載 LTS 版（頁面上寫著「建議大多數使用者使用」的那個）
  2. 執行安裝，一樣全部預設值下一步
  3. 裝完重開終端機，確認：
確認 Node.js 裝好了
node --version

看到 v22.x.x 之類的版本號就 OK（v20 以上都可以）。

1-4. 告訴 Git 你是誰（只要做一次）

Git 每次記錄都會署名。把下面兩行的名字和 email 換成你自己的再貼上（email 建議跟等一下註冊 GitHub 的相同）：

設定署名（記得改成你的資料）
git config --global user.name "你的名字"
git config --global user.email "你的email@example.com"
2

把專案抓到你的電腦

🎯 目標：擁有自己的 Collector 副本，並在電腦上準備就緒。

2-1. 註冊 GitHub 並 Fork

  1. 到 github.com/signup 註冊帳號（帳號名稱會出現在你的網站網址裡，建議取短一點的英文）。已經有帳號就直接下一步。
  2. 登入後打開 github.com/Sallynium/Collector，點右上角 Fork → 按綠色 Create fork。「Fork」= 複製一份到你帳號底下，從此它完全是你的。

2-2. Clone：下載到電腦

開終端機，貼上下面的指令，把「你的帳號」換成你的 GitHub 帳號名稱。這會在你目前的位置建立一個 Collector 資料夾：

下載專案（記得改帳號名）
git clone https://github.com/你的帳號/Collector.git
cd Collector
終端機打開時通常位在你的「使用者資料夾」（例如 C:\Users\你的名字），專案就會下載到那裡。想放別的地方，先用 cd 移動過去再執行，例如 cd D:\projects。

2-3. 安裝專案依賴

接著讓 Node.js 把專案需要的零件裝齊（第一次要跑幾分鐘，之後不用再做）：

安裝依賴（在 Collector 資料夾裡執行）
npm install

跑完出現一些 added xxx packages 的字樣就完成了。有黃色 warning 是正常的，不用理。

3

啟動後台，整理你的收藏

🎯 目標：在自己電腦上把娃娃上架、去背、拼場景。這是最好玩的一步。

3-1. 啟動本機網站

啟動（在 Collector 資料夾裡執行）
npm run dev
  1. 等它顯示 Ready 後，用瀏覽器打開 http://localhost:3000
  2. 右上角直接就有「管理後台」按鈕——在自己電腦上不需要登入，因為只有你摸得到
  3. 這個終端機視窗保持開著；想關掉網站時按 Ctrl + C

3-2. 建立分類和娃娃檔案

  1. 進入 庫存管理，先在「系列」和「收藏櫃」分頁各建至少一筆（例如系列「戰隊限定」、收藏櫃「A 櫃」）
  2. 回「娃娃」分頁新增你的第一隻娃娃，選好系列和櫃位

3-3. 上傳照片，AI 自動去背

  1. 點進娃娃 → 新增造型 → 選一張娃娃照片（背景越乾淨效果越好）
  2. AI 會在你自己的瀏覽器裡去背（第一次要下載模型，等 1–2 分鐘；照片不會被上傳到任何地方，隱私滿分）
  3. AI 誤刪耳機、頭髮？用 🟢 綠色筆刷塗回來；背景有殘留就用 🔴 紅色筆刷擦掉
  4. 取個造型名稱按儲存。一隻娃娃可以存多個造型（預設、夏季穿搭、戰隊服…），展示時可以切換

3-4. 拼場景、發布場景

  1. 後台進 虛擬收藏室 → 建立場景 → 上傳背景圖（書櫃照、電競桌照都行）
  2. 把娃娃「放入場景」，拖曳移動、四角控點縮放旋轉、調整圖層順序 → 儲存佈局
  3. 回場景列表按 發布，這個場景才會出現在對外的展示頁
你做的所有編輯都存在專案資料夾裡：清單在 content/data.json，去背圖在 public/collection/。好奇可以打開看看，不小心改壞也沒關係——下一步的 git 會保護你。
4

發布上網（第一次需要多做一個設定）

🎯 目標：把整理好的收藏推上 GitHub，拿到你的公開網址。

4-1. 開啟 GitHub Pages（只做這一次）

  1. 打開你的 github.com/你的帳號/Collector → 上方 Settings → 左邊選單 Pages
  2. 「Build and deployment」的 Source 下拉選單改成 GitHub Actions，這樣就好

4-2. 推上去！

回到終端機（如果 npm run dev 還在跑，先按 Ctrl + C 停掉），貼上這三行：

發布收藏（在 Collector 資料夾裡執行）
git add -A
git commit -m "更新收藏"
git push
  1. 第一次 push 會跳出 GitHub 登入視窗，點「Sign in with your browser」用瀏覽器授權即可（之後就不會再問）
  2. push 完，到你的 GitHub 專案頁點上方 Actions 分頁，會看到一個工作正在轉圈圈，等它變成 綠色勾勾（約 2–3 分鐘）

4-3. 打開你的網站 🎉

你的網址是（把「你的帳號」換掉，注意全部小寫）：

https://你的帳號.github.io/Collector/

把這個網址分享給朋友吧！網站上沒有任何管理入口——訪客只能欣賞，不可能改到你的收藏。

之後的日常：5 分鐘更新流程

🎯 目標：記住這一套循環，以後買了新娃娃就照做。

  1. 開終端機，進到專案資料夾，啟動後台：
① 啟動後台
cd Collector
npm run dev
  1. 瀏覽器開 localhost:3000，上架新娃娃 / 改場景，改到滿意
  2. 回終端機按 Ctrl + C 停掉，然後發布：
② 發布

  1. 等 Actions 綠勾（2–3 分鐘），網站就是最新版了 ✨
commit 訊息（引號裡的字）可以自由發揮，例如 "新增 Faker 簽名娃娃"——這會變成你的收藏更新日誌，以後回頭看很有味道。

常見問題

1. push 的時候被拒絕（rejected / failed to push）

通常是 GitHub 上有你電腦裡沒有的變動（例如你在別台電腦推過）。先拉再推：

先同步再發布
git pull --rebase
git push
2. push 了但網站沒更新

到 GitHub 專案頁的 Actions 分頁看最新那筆：還在轉圈圈就再等等；如果是紅色叉叉，點進去看錯誤訊息（截圖下來問人也行）。另外瀏覽器可能有快取，按 Ctrl + F5 強制重新整理。

3. npm run dev 起不來 / 一片紅字

依序檢查：①終端機目前位置是不是在 Collector 資料夾裡（可以打 cd Collector 再試）②有沒有先跑過 npm install ③ node --version 是不是 v20 以上。都對還是不行，把紅字的最後十行截圖下來查。

4. 照片很大張，會不會塞爆？

去背後存的 PNG 通常只有幾百 KB，100 隻娃娃大約幾十 MB，離 GitHub 的建議上限（1GB）還很遠。要注意的是場景背景圖：上傳前建議先縮到 2000px 寬以內。單一檔案不能超過 100MB（正常照片不可能超過）。

5. 我可以改網站標題、換配色嗎？

可以！這整個專案都是你的了。標題文字在 src/app/layout.tsx 和 src/app/page.tsx，用任何文字編輯器（推薦 VS Code）打開改字就行，改完照常 push。改壞了就 git checkout . 全部復原。

6. 原版 Collector 之後更新了，我要怎麼跟上？

打開你的 GitHub 專案頁，如果上方出現「This branch is X commits behind」，點 Sync fork → Update branch，然後回電腦執行 git pull 把更新拉下來。你的收藏資料不會受影響。

7. 換了新電腦怎麼辦？

完全不怕——你的整個收藏（包括圖片）都在 GitHub 上。新電腦照步驟 1 裝好 Git 和 Node.js，然後 git clone + npm install，一切如舊。

8. 不想要某些東西被看到？

場景只有按過「發布」的才會出現在網站上，草稿場景訪客看不到。但注意：repo 是公開的，照片檔案本身在 GitHub 上看得到。如果整個收藏都不想公開，可以把 repo 設為 Private——不過免費帳號的 GitHub Pages 只支援公開 repo，這時就要考慮改用 Vercel 部署（進階主題）。
